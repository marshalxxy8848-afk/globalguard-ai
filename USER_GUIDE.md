## 1. 如何启动系统

### 第一步：打开终端

在电脑上找到 **终端**（Terminal）或 **命令提示符**（CMD）：

- Windows：按 `Win` 键，输入 `cmd`，回车
- Mac：按 `Command + 空格`，输入 `终端`，回车

### 第二步：进入项目目录

```bash
cd D:\globalguard-ai
```

### 第三步：启动开发服务器

```bash
npm run dev
```

你会看到类似这样的输出：

```
▲  Next.js 16.2.6
- Local: http://localhost:3456
```

这表示服务器已经启动成功了。

### 第四步：打开浏览器

打开 Chrome 或 Edge，在地址栏输入：

```
http://localhost:3456
```

你就看到 GlobalGuard AI 的主界面了。

> ⚠️ **关掉终端 = 关掉服务器**。每次使用都要重复上面的步骤。如果你想关闭服务器，按 `Ctrl + C` 即可。

## 2. 注册 & 登录

### 注册账号

1. 点击右上角的 **Sign up** 按钮
2. 输入你的邮箱（随便一个真实邮箱即可，不需要验证）
3. 输入密码（至少 6 位）
4. 点击 **Create Account**
5. 注册成功后自动跳转到首页，右上角会显示你的邮箱

### 登录

1. 点击右上角的 **Log in**
2. 输入注册的邮箱和密码
3. 点击 **Log In**

### 登出

点击右上角的 **Log out** 按钮即可。

> 💡 **不登录也能用**。不上传产品照片做审计是免费的，不需要账号。登录后系统会自动保存你的审计历史。

## 3. 上传产品照片做审计

这是 GlobalGuard AI 的核心功能。

### 操作步骤

1. 在首页中间的大虚线框上 **点击** 或 **拖拽** 一张产品照片进去
2. 系统自动开始分析：
   - 第一步：上传图片
   - 第二步：AI 识别产品（Vision Classifier）
   - 第三步：查找 HS 编码（Rule Retriever）
   - 第四步：做合规审计（Compliance Auditor）
3. 几秒钟后，你会看到一份完整的审计报告：

   **报告包括：**
   - **产品名称** — AI 识别出的产品
   - **HS 编码** — 海关编码 + 描述
   - **风险等级** — LOW（低）/ MEDIUM（中）/ HIGH（高）
   - **美国关税评估** — 预估关税金额、税率、T86 影响
   - **欧盟关税 + VAT 评估** — 预估关税金额、VAT 金额、总金额
   - **建议申报描述** — 给报关行看的英文描述
   - **风险提示** — 是否需要特别注意

### 支持的文件
- 格式：PNG、JPG
- 大小：不超过 5MB

### 关于 AI 识别准确度

> ⚠️ 当前使用 **Mock 模式**（因为没有配置真实的 AI API Key），AI 识别结果是模拟数据，仅供参考。配置真实 Key 后准确度会大幅提升（见第 7 节）。

**前提：你已经登录了账号。**

2. 你会看到一个列表，展示所有历史审计记录

## 5. 下载 PDF 报告

每次完成审计后，报告下方会有一个 **Download PDF Report** 按钮：

1. 完成审计，看到完整报告
2. 向下滚动，点击 **Download PDF Report**
3. 浏览器会自动下载一份 A4 格式的 PDF

**PDF 包含：**
- 产品信息（名称、HS 编码、描述）
- 美国关税详情
- 欧盟关税 + VAT 详情
- 建议报关描述
- 风险警告

## 6. OpenClaw 自动监控是什么

### 一句话解释

OpenClaw 是 GlobalGuard AI 内置的 **自动化政策监控机器人**。它会每隔 6 小时自动上网检查美国海关政策有没有变化（比如 T86 免税政策、Section 321、针对中国商品的关税调整）。

### 它是怎么工作的

你不需要手动操作，系统已经设置好了。流程如下：

```
每 6 小时 → 自动搜索 CBP 官网和新闻 → 有变化？→ 记录到系统
                                   → 没变化？→ 记录"无变化"
```

### 监控什么内容

- **Section 321 / T86** — 美国 800 美元以下包裹的免税政策
- **中国关税** — 针对中国商品的额外关税
- **$800 豁免额** — 最低限度条款的变化

### 在哪里查看状态

在 Claude Code 左侧边栏找到 **Scheduled Tasks**：

- 如果看到 ✅ green checkmark，说明正常运行
- 点击可以查看最近一次检查的时间和结果
- 每次检查后，你会在当前对话中收到通知

### 我可以修改或者关闭它吗？

可以。如果你想让监控机器人暂停，使用这个命令：

```
⏸️ 暂停：告诉 Claude "暂停政策监控" 即可
▶️ 恢复：告诉 Claude "恢复政策监控"
❌ 删除：告诉 Claude "删除 trade-policy-monitor 任务"
```

## 7. Mock 模式 vs 真实 AI 模式

### 当前状态：Mock 模式

现在项目的 `.env.local` 文件中配置了你的 DeepSeek Key，但 DeepSeek 模型**不支持图片输入**（只能处理文字），所以系统运行在 **Mock（模拟）模式**：

- AI 视觉识别返回的是示例数据（比如 "Wireless Bluetooth Headphones"）
- 关税计算用的是本地数据库的默认值
- **不需要任何费用，适合先体验功能**

### 配置真实 AI Key（可选）

如果你想用真正的 AI 识别能力，需要配置 **OpenAI** 或 **Anthropic** 的 Key（因为 DeepSeek 不支持图片识别）：

1. 打开项目根目录下的 `.env.local` 文件
2. 找到这一行：
   ```
   OPENAI_API_KEY=
   ```
3. 填入你的 OpenAI API Key
   ```
   OPENAI_API_KEY=sk-your-api-key-here
   ```
4. 保存文件，重启服务器（按 `Ctrl + C` 再重新 `npm run dev`）

> 💡 **关于 DeepSeek Key**：你的 DeepSeek Key 已经配置好了，虽然 DeepSeek 不能做图片识别，但如果未来 DeepSeek 支持了视觉功能，系统会自动优先使用它。目前建议配置 OpenAI Key 以获得真正的 AI 视觉分析。

### 三种模式对比

| 功能 | Mock 模式 | 有 DeepSeek + 无视觉 | 有 OpenAI/Anthropic Key |
|------|-----------|---------------------|------------------------|
| 产品识别 | 示例数据 | 示例数据（DeepSeek 不支持图片） | 真实 AI 识别 |
| HS 编码匹配 | 关键词检索 | 关键词检索 | AI + 数据库双重匹配 |
| 关税计算 | 基于数据库 | 基于数据库 | 基于数据库 |
| PDF 导出 | ✅ | ✅ | ✅ |
| 历史记录 | ✅ | ✅ | ✅ |
| 费用 | 免费 | 免费 | 按 API 调用量收费 |
| 产品识别 | 示例数据（模拟的） | 真实 AI 识别 |
| HS 编码匹配 | 关键词检索 | AI + 数据库双重匹配 |
| 关税计算 | 基于数据库 | 基于数据库 |
| PDF 导出 | ✅ | ✅ |
| 历史记录 | ✅ | ✅ |
| 费用 | 免费 | 按 API 调用量收费 |

> 💡 **建议**：先体验 Mock 模式，确定这个工具对你有用，再配置真实 API Key。

## 8. 常见问题

### Q: 启动时提示 "port 3456 already in use"

说明端口被占用了。可以改端口号：

```bash
# 在终端中，先按 Ctrl + C 停掉之前的进程
# 然后换个端口启动
npx next dev -p 3457
```

然后在浏览器打开 `http://localhost:3457`

### Q: 数据库出问题了，想重置

数据库文件在 `D:\globalguard-ai\prisma\dev.db`。如果你想清空所有数据：

```bash
# 1. 删除数据库文件
del D:\globalguard-ai\prisma\dev.db

# 2. 重新创建
cd D:\globalguard-ai
npx prisma db push
```

> ⚠️ 这会删掉所有用户账号和历史审计记录！

### Q: 上传图片后一直转圈没反应

1. 检查图片是否超过 5MB
2. 检查终端窗口有没有报错
3. 按 `F12` 打开浏览器开发者工具 → 看 Console 有没有错误
4. 如果还是不行，试试重启服务器

### Q: 登录后 History 页面显示 401

说明登录状态丢失了。尝试：
1. 登出
2. 重新登录
3. 再访问 History

### Q: 这些数据准确吗？

> ⚠️ **免责声明**：GlobalGuard AI 是一个辅助工具，提供的是参考数据，**不是法律文件**。实际的 HS 编码归类、关税税率需要以报关行或海关的最终裁定为准。建议将报告作为初步参考，然后咨询专业的报关行。

## 附录：技术架构（给好奇的你）

```
前端 (Next.js)  ←→  API 路由  ←→  Prisma / SQLite (数据库)
                    ↕
               AI Vision (OpenAI/DeepSeek/Anthropic)
                    ↕
               HS Code 数据库 (52 个编码, 9 大类)
                    ↕
               OpenClaw (每 6 小时监控政策变化)
```

- **前端**：React + Tailwind CSS（深色主题）
- **后端**：Next.js API Routes
- **数据库**：SQLite（文件存储在 `prisma/dev.db`）
- **AI**：支持 OpenAI、DeepSeek、Anthropic
- **认证**：JWT + HTTP-only Cookie
- **监控**：Claude Code Scheduled Tasks（每 6 小时自动运行）
